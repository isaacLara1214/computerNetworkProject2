Project Overview

This project was created by Isaac Lara for Computer Networking. It is hosted publicly and shows how websites use domain names, IP addresses, and secure communication over the internet.

Networking Concepts

DNS

DNS translates a human-readable domain name into an IP address so browsers can locate the correct web server.

IP Addressing

Devices on the internet communicate using IP addresses. This site can resolve to both IPv4 and IPv6 addresses.

Hosting

This website is hosted publicly so users can access it from a web browser anywhere with an internet connection.

HTTP vs HTTPS

HTTP

HTTP is the protocol used to transfer web content between clients and servers.

  • Transfers website data
  • Uses request/response model
  • Can return status codes like 200 OK

HTTPS

HTTPS is the secure version of HTTP. It encrypts communication using TLS to protect data between the client and server.

  • Encrypts traffic
  • Improves security and trust
  • Required for modern public websites

Security Features

Current Status

HTTPS Enabled

This site uses secure communication, which helps protect data in transit between the browser and the server.

HTTPS / TLS

All traffic is encrypted end-to-end via Netlify's automatic TLS certificate. HTTP requests are force-redirected to HTTPS.

Security Headers

HSTS, Content-Security-Policy, X-Frame-Options, X-Content-Type-Options, and Referrer-Policy headers are set on every response.

XSS Protection

All user-submitted content is sanitized with DOMPurify before being rendered in the DOM, preventing script injection attacks.

Rate Limiting

The guestbook enforces a 30-second cooldown between submissions to prevent abuse and flooding.

Firestore Rules

Server-side security rules validate every write — enforcing field types, length limits, and blocking all updates and deletes.

Input Validation

All form inputs are validated for length and content on both the client and server before any data is stored.

Guestbook

Leave a message — stored securely in Firebase Firestore.
